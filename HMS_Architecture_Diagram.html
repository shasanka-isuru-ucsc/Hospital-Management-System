🐇 RabbitMQ Event Bus — Topic / Direct / Fanout Exchanges
Management UI · Durable queues · Dead letter queues · Message persistence
billing.opd → Finance Service billing.wound → Finance Service billing.ward → Finance Service billing.lab → Finance Service billing.pharmacy → Finance Service lab.requested → Lab Service queue.updated → WebSocket Bridge → Display pharmacy.new_rx → Pharmacy Display notify.appointment → Notification Service notify.queue → Notification Service
⚡ Socket.IO (embedded in API Gateway)
Subscribes to RabbitMQ fanout → pushes real-time updates to connected browser clients (Queue Display, Pharmacy Display) via WebSocket. No polling needed. Redis stores queue state for reconnecting clients.
📦 Event Guarantees
Durable queues + persistent messages. If Finance Service is down, billing events queue up and are processed on recovery. At-least-once delivery with idempotent handlers. DLQ for failed messages.

Primary Databases
🐘 PostgreSQL 15
Relational ACID Database
users · patients · doctors · appointments · tokens · invoices · billing_line_items · wards · beds · admissions · lab_orders · departments · schedules · nurse_allocations · staff

Used by: Auth, Reception, Staff, Finance, Ward, Lab (relational data), Notification
🍃 MongoDB 7
Document Database — Clinical Records
clinical_sessions { vitals, diagnoses, prescriptions[], wound_details, scan_images[], report_requests[] } · patient_health_history { vitals_history[], medical_history[], medications[], goals[] }

Used by: Clinical Service, Lab Service (flexible schema)
⚡ Redis 7
In-Memory Cache + Session Store
JWT session blacklist · Queue token state (WAITING/CALLED/DONE) · Appointment slot locking (prevent double-booking) · Doctor availability cache · Refresh token tracking · Real-time queue snapshot for reconnecting WS clients

Used by: API Gateway, Reception Service
File & Object Storage
📦 MinIO (S3-compatible)
Object Storage — Self-hosted
hms-avatars/ (doctor + patient photos) · hms-scan-documents/ (OPD diagnosis images) · hms-wound-images/ (wound care photos) · hms-lab-reports/ (lab report PDFs) · hms-prescriptions/ (external Rx PDFs) · hms-doctor-banners/

Access via pre-signed URLs (15-min expiry). S3-compatible → easy AWS S3 migration.
🌐 DNS + SSL
Nginx + Let's Encrypt / cert-manager
TLS 1.2+ enforced on all external endpoints. Internal service traffic over private Docker/k8s network (HTTP). Nginx manages certificate auto-renewal. All external ports: 80 (redirect) + 443 only.
Deploy
Deployment & Observability
🐳 Docker → Kubernetes
Dev: Docker Compose · Prod: K8s + Helm
Each service: Deployment (2+ replicas) + ClusterIP. HPA on all stateless services. Rolling update strategy with auto-rollback. Secrets via k8s Secrets / HashiCorp Vault.
⚙️ CI/CD Pipeline
GitHub Actions
Push → lint → Jest unit tests → Trivy security scan → Docker build → push to registry → deploy staging → smoke tests → tagged release → kubectl rollout prod.
📊 Monitoring
Prometheus + Grafana + ELK
Each service exports /metrics. Prometheus scrape every 15s. Grafana dashboards per service. Winston structured JSON logs → Elasticsearch → Kibana. Alerting on p95 latency > 300ms.
🔒 Backup Strategy
Automated daily backups
pg_dump daily → MinIO/S3 cold bucket. mongodump daily. Redis RDB snapshot hourly. MinIO replication to secondary region. 30-day retention. Tested restore quarterly.
Key Service Interactions
How services talk to each other — sync (HTTP) vs async (RabbitMQ)
RECEPTION → FINANCE
🟡 Async: Patient registration triggers billing account creation event
🟡 Async: Appointment cancellation → refund event
CLINICAL → FINANCE
🟡 Async: Session complete → billing.opd / billing.wound event
🔵 Sync: Doctor applies discount → PUT /finance/invoices/:id/discount
CLINICAL → LAB
🟡 Async: Doctor requests lab test → lab.requested event → Lab creates placeholder order
🔵 Sync: GET /lab/orders?patient_id= to view results
LAB → FINANCE
🟡 Async: Lab order payment → billing.lab event to Finance
🔵 Sync: Lab Tech records payment via PUT /lab/orders/:id/payment
WARD → FINANCE
🟡 Async: Patient discharge → billing.ward event → Finance closes ward invoice
🔵 Sync: Admin views ward billing stack → GET /ward/admissions/:id/services
AUTH → ALL SERVICES
🔵 Sync: Admin creates doctor → Staff Service calls Auth Service internally to create user account
Gateway validates JWT, injects headers — services never call Auth directly
CLINICAL → PHARMACY DISPLAY
🟡 Async: Doctor saves internal prescription → pharmacy.new_rx event → WebSocket push → Pharmacy Display updates in real time
RECEPTION → QUEUE DISPLAY
🟡 Async: Token status change → queue.updated fanout → WebSocket bridge → Queue Display boards refresh instantly. Redis holds snapshot for reconnects.
🟡 Async via RabbitMQ
🔵 Sync HTTP via API Gateway
